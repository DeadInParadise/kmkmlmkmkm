Learning Roadmap

Topics in exact order — each one builds on the previous. Don't skip.
01

Print & Variables

Foundation of everything. Learn how Python outputs text, stores data in variables, and handles basic data types (int, float, str, bool).

print() variables data types 4 questions
02

Input & Type Conversion

Make your programs interactive. Learn input(), and why you must convert types — input always returns a string, even if user types a number.

input() int() float() str() 3 questions
03

If / Elif / Else

Decision making. Programs that can choose different paths based on conditions. Master comparison operators and boolean logic here.

if / elif / else == != > < and or 5 questions
04

Loops — for & while

Repetition and automation. for loops with range(), while loops with conditions, break and continue. This is where programming starts feeling powerful.

for / while range() break continue 7 questions
05

Functions

Reusable blocks of code. def, parameters, return values, and scope. Good functions make you write less code and avoid repeating yourself.

def / return parameters / scope recursion 6 questions
06

Strings

Text manipulation. Slicing, methods like .split() .join() .replace(), and string formatting. Most real programs deal heavily with text.

slicing string methods f-strings 5 questions
07

Lists

Ordered collections of data. Indexing, slicing, append/remove, and looping over lists. Most common data structure you'll use daily.

lists / indexing append remove sort list comprehension 6 questions
08

Dictionaries & Sets

Key-value storage and unique collections. Essential for counting, grouping, and lookups. Much faster than lists for searching by name.

dict / set .items() .keys() .get() 4 questions
09

File Handling

Read from and write to files on disk. open(), with statements, and modes ('r','w','a'). Makes your programs actually save data between runs.

open() / with read write append 3 questions
10

Mini Projects

Combine everything. Build real, working programs from scratch. This is where concepts click — when you build something end-to-end.

Recommended Books

Read in this order. Don't buy all at once — finish one, then move.
Start Here — Book 1

Python Crash Course

by Eric Matthes · 3rd Edition

The best beginner book. Teaches concepts cleanly, then immediately gives you exercises. Part 2 has three full projects: a game, data visualizations, and a web app.

  • Has Try-It-Yourself exercises after every section
  • Three end-of-book projects to build real things
  • Same topic order as this roadmap
  • Doesn't waste pages on theory — gets to code fast
How to use: Read one chapter → do book exercises → do matching questions from this app.
Book 2 — After Functions

Python Workout

by Reuven M. Lerner

Pure exercise book — exactly what you asked for. 50 problems, each with the question, hints, full solution, and explanation of why that approach works.

  • Problem → Hint → Solution → Explanation format
  • Tells you what to think about before you start
  • Covers strings, lists, dicts, files, functions deeply
  • Closest book to "questions + advice + answers"
When: Start after finishing Loops & Functions in PCC.
Book 3 — Real World

Automate the Boring Stuff

by Al Sweigart Free Online

Practical, project-focused. Teaches you to automate Excel, PDFs, web scraping, scheduling, and more. No fluff — just "here's a problem, here's how to solve it with Python."

  • Free at automatetheboringstuff.com
  • Real-world projects you'll actually use
  • Great for motivation — builds things you can show people
  • Each chapter is standalone, pick what interests you
automatetheboringstuff.com ↗
Bonus — Problem Book

Exercises for Programmers

by Brian P. Hogan

57 progressively harder problems. Language-agnostic, you solve in Python. Uniquely tells you what constraints to think about before you code — good for developing problem-solving instincts.

  • Gives you thinking prompts, not just problems
  • Good bridge between beginner and intermediate
Study Tips
🔁

Read Then Code Immediately

Don't read a whole chapter passively. Read one section, close the book, type the code from scratch. Typing it yourself is what builds memory.

🚫

Don't Copy-Paste Solutions

If you're stuck, look at the solution to understand it, then close it and re-write it without looking. That's the only way it sticks.

⏱️

1 Hour Daily > 8 Hours Weekend

Consistency beats intensity. 1 hour every day for 3 months is worth more than studying 8 hours on weekends only.

🐛

Read Error Messages Fully

Most beginners ignore the error. Python's errors tell you exactly what went wrong and which line. Train yourself to read them before Googling.

📓

Keep a Code Journal

Write down one new thing you learned each session in your own words. Forces you to actually understand it, not just recognize it.

🌐

HackerRank for Extra Drills

After finishing loops and functions, do the Python Easy track on HackerRank.com. Free, graded, and has hundreds of problems ordered by difficulty.
